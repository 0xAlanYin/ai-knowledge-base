# Documentation Update Commit Template

## Template Description
For commits that update documentation, comments, README, or other documentation content.

## Commit Format
```
docs: <brief description>

- <documentation update 1>
- <documentation update 2>
- <documentation update 3>

<optional: related issue or PR>
```

## Field Description
- **brief description**: Under 50 words, describes the documentation update
- **documentation updates**: Start with verbs, describe specific updates

## Examples

### Example 1: Update Project Documentation
```
docs: update AGENTS.md with practical experience

- add tool selection experience summary
- document problem-solving patterns
- update version to 1.1.0

Closes #31
```

### Example 2: Update API Documentation
```
docs: update API usage examples

- add code examples for new endpoints
- improve parameter descriptions
- fix outdated documentation

Related to #18
```

### Example 3: Update Configuration Guide
```
docs: improve configuration guide

- add environment variables documentation
- update deployment instructions
- fix broken links
```

## Best Practices
1. **Accuracy**: Ensure documentation accurately reflects code state
2. **Completeness**: Provide complete examples and explanations
3. **Readability**: Use clear and understandable language
4. **Timeliness**: Update documentation promptly after code changes
